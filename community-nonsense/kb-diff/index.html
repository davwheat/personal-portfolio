      • Important questions with detailed answers

      • What's in this data?

        This data contains:

        • the article's ID (a number assigned to it)
        • the article content in JSON format
        • the article's URL
        • the article's category, category URL and category ID (the number assigned to the article's category)
        • the article's SEO metadata (description)
        • the article's visibility state (true/false)

        No personal information is transferred in this request.

      • Why do I have to get this data myself?

        For security purposes, requests to the giffgaff API can only be made from within the "giffgaff.com" domain, or directly in a user's browser. This is to prevent malicious sites accessing the "me" API which provides your address, full name, and other personal info.

        It's possible to download these server-side but that would require infrastructure which I honestly can't be bothered to support, run, or pay for (if needed). I also can't maintain a list of every article 24/7. It requires far more effort than it's worth.
